# Contributing to Web Frontend

Thank you for your interest in contributing to the TechCorp Web Frontend project! This guide outlines the process and guidelines for contributing.

## Getting Started

1. **Fork the repository** and clone it locally.
2. Install dependencies with `npm install`.
3. Create a feature branch from `main`: `git checkout -b feature/my-feature`.
4. Make your changes and ensure all tests pass with `npm test`.
5. Push your branch and open a Pull Request.

## Code Style

- Follow the existing ESLint and Prettier configurations.
- Use TypeScript for all new files.
- Write descriptive commit messages following [Conventional Commits](https://www.conventionalcommits.org/).

## Pull Request Guidelines

- Keep PRs focused on a single concern.
- Include a clear description of the changes.
- Add or update tests as appropriate.
- Ensure CI passes before requesting review.
- Request at least one review from a team member.

## Reporting Issues

- Use GitHub Issues to report bugs or request features.
- Include steps to reproduce, expected behavior, and actual behavior for bug reports.
- Label issues appropriately using the available labels.

## Code Review

All contributions require review. Reviewers should:

- Verify correctness and test coverage.
- Check for adherence to coding standards.
- Provide constructive, actionable feedback.

## Questions?

Reach out to the platform team on Slack at `#platform-engineering` for any questions.
